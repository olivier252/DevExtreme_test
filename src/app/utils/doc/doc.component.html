DX inclut une couche de données qui est un set de composants qui permet de lire et écrire des données.

2 catégories d'objet pour cela :

  • Le DataSource est un objet stateful qui garde le tri, le regroupement, le filtrage, et conserve les propriétés de transformation des données et les applique chaque fois que les données sont chargées. Il fournit également des événements destinés à gérer les données changeantes et l’état. Le DataSource est un objet qui fournit une API pour le traitement des données à partir d’un magasin sous-jacent.
  • Les Stores : contiennent la logique d’accès aux données sous-jacente de DataSource. Ils implémentent une interface universelle. Chaque store contient le même set de méthodes

3 types de stores selon les scénarios d'accès aux datas les plus répandus :

  • ArrayStore
  • LocalStore
  • ODataStore

Toutes les opérations de transfert de données dans la couche de données sont asynchrones, que ce soit un accès à un service distant ou à des données locales. Il est conçu pour prendre en charge l’interface Store universelle, quelle que soit la source de données auxquelles vous accédez. Ces opérations renvoient une Promesse native ou une jQuery.Promise lorsque vous utilisez jQuery
